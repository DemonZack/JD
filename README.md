### 智能客服系统（内部创新项目） · 核心研发  
**时间：2023.06 – 2023.11 · 聚名集团**

内部自研的一套 **大模型 + 企业知识库** 的智能客服系统，用于承接公司多条产品线的售前 / 售后咨询。

- **定位与目标**：在不明显增加客服人力的情况下，消化持续增长的咨询量，并保证答复的一致性与专业度。
- **打通“文档 → 知识 → 回答”链路**：围绕公司现有的产品文档、FAQ、操作手册，构建了上传、解析、索引、检索、回答的全流程，使文档从“静态存档”变成“可被实时调用的知识资产”。
- **服务多角色场景**：同时覆盖客服、销售、实施等角色的需求，支持常见问题自动应答，也支持复杂问题由客服接管，并完整保留上下文。
- **业务侧收益**：系统在数月内完成试点和推广，上线后能自动处理大部分标准化问题，让人工客服把时间集中在真正复杂的场景上，人效明显提升。

## 开源与技术方向

- **simplejrpc**：轻量级 RPC 框架，用于在内部系统中快速搭建稳定、可观测的服务间调用通道。  
  GitHub：`https://github.com/GMSSH/app-sdk-py`
- **simplejrpc_gfile**：文件服务 SDK，简化存储服务接入成本。  
  PyPI：`https://pypi.org/project/simplejrpc-gfile/`
- **gmscaffold**：项目脚手架工具，用于在团队内部统一工程结构和基础设施，降低新项目启动成本。  
  PyPI：`https://pypi.org/project/gmscaffold/`
- **sentrylog**：与 Sentry 集成的日志组件，让团队在不改动大量业务代码的前提下，引入统一的异常上报通道。  
  PyPI：`https://pypi.org/project/sentrylog/`
- **simplejrpc-gbox**：通用工具集合，沉淀日常开发中高复用的小能力，避免“在各个项目间复制粘贴”。  
  PyPI：`https://pypi.org/project/simplejrpc-gbox/`
- **simplejrpc-go**：基于 Go 的 simplejrpc 实现，方便在多语言技术栈中统一 RPC 协议。  
  GitHub：`https://github.com/DemonZack/simplejrpc-go`
- **trpc-agent-go 贡献者**：参与 AI 智能体框架 `trpc-agent-go` 的开发，为内部与社区项目探索“Agent + 业务系统”的结合方式。  
  GitHub：`https://github.com/trpc-group/trpc-agent-go`
